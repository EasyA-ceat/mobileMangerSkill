# 手机控制技能包接口规范
## 一、平台侧HTTP接口（OpenClaw框架调用）
统一前缀：`/api/phone-control/v1`，请求/响应均为JSON格式。
| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 响应示例 |
|----------|----------|----------|----------|----------|
| `/device/list` | POST | 获取已连接设备列表 | 无 | <pre>{<br>  "code": 0,<br>  "message": "success",<br>  "data": [<br>    {"device_id": "emulator-5554", "name": "小米13", "status": "connected", "type": "usb"}<br>  ]<br>}</pre> |
| `/device/connect` | POST | 连接设备 | <pre>{"device": "192.168.1.100:5555"}</pre> | <pre>{<br>  "code": 0,<br>  "message": "连接成功",<br>  "data": {"device_id": "192.168.1.100:5555", "name": "华为Mate 40 Pro"}<br>}</pre> |
| `/device/disconnect` | POST | 断开设备 | <pre>{"device_id": "192.168.1.100:5555"}</pre> | <pre>{"code":0, "message":"断开成功"}</pre> |
| `/control/click` | POST | 点击操作 | <pre>{"device_id": "xxx", "target": "500,1000" / "设置"}</pre> | <pre>{"code":0, "message":"点击成功"}</pre> |
| `/control/swipe` | POST | 滑动操作 | <pre>{"device_id": "xxx", "start": "100,2000", "end": "100,500", "duration": 500}</pre> | <pre>{"code":0, "message":"滑动成功"}</pre> |
| `/control/input` | POST | 输入文字 | <pre>{"device_id": "xxx", "text": "测试内容"}</pre> | <pre>{"code":0, "message":"输入成功"}</pre> |
| `/control/key` | POST | 系统按键 | <pre>{"device_id": "xxx", "key": "home"}</pre> | 支持按键：back/home/power/volume_up/volume_down/menu |
| `/control/app` | POST | 启动/关闭App | <pre>{"device_id": "xxx", "action": "start", "package": "com.tencent.mm"}</pre> | <pre>{"code":0, "message":"App启动成功"}</pre> |
| `/screen/screenshot` | POST | 截屏 | <pre>{"device_id": "xxx", "save_path": "/tmp/1.png"}</pre> | 返回Base64图片或保存路径 |
| `/script/run` | POST | 执行操作脚本 | <pre>{"device_id": "xxx", "script": [{"action": "key", "params": {"key": "home"}}]}</pre> | <pre>{"code":0, "message":"脚本执行成功"}</pre> |

## 二、用户侧指令接口（聊天中直接调用）
| 指令格式 | 功能说明 | 示例 |
|----------|----------|------|
| `手机列表` / `phone list` | 列出已连接设备 | `手机列表` |
| `连接手机 <设备IP/序列号>` | 连接设备 | `连接手机 192.168.3.105` |
| `断开手机 <设备ID>` | 断开设备 | `断开手机 192.168.3.105` |
| `点击 <坐标/文字>` | 点击操作 | `点击 设置` / `点击 500,1200` |
| `滑动 <起点> <终点> [时长]` | 滑动操作 | `滑动 100,2000 100,500` |
| `输入 <文本内容>` | 输入文字 | `输入 测试内容123` |
| `按键 <按键名>` | 系统按键 | `按键 返回` / `按键 home` |
| `打开App <App名称/包名>` | 启动App | `打开App 微信` |
| `关闭App <App名称/包名>` | 关闭App | `关闭App 淘宝` |
| `截屏 [保存路径]` | 截屏 | `截屏` / `截屏 /tmp/1.png` |
| `执行脚本 <脚本内容>` | 执行操作脚本 | `执行脚本 刷朋友圈` |

## 三、错误码定义
| 错误码 | 含义 | 说明 |
|--------|------|------|
| 0 | 成功 | 操作正常完成 |
| 1001 | 设备未找到 | 指定设备不存在或未连接 |
| 1002 | 设备连接失败 | 检查ADB权限、网络连接 |
| 1003 | 操作执行失败 | 指令执行失败，检查参数或设备状态 |
| 1004 | 目标未找到 | 点击的文字未在屏幕识别到 |
| 1005 | 权限不足 | 缺少设备调试授权 |
| 2001 | 参数错误 | 请求参数缺失或格式错误 |
| 5000 | 系统内部错误 | 服务异常，查看日志 |
