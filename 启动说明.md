## 🔐 安全配置（重要！）

### 1. 配置环境变量

复制 `.env.example` 为 `.env` 并编辑：

```bash
# Windows
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

编辑 `.env` 文件：
```env
# 环境类型
ENV=development

# 生成安全的API Key（重要！）
# 生成方式: python -c "import os; print(os.urandom(32).hex())"
API_KEY=your-generated-secure-api-key

# 服务配置
HOST=0.0.0.0
PORT=8080

# CORS允许的来源
ALLOWED_ORIGINS=http://localhost:8080,http://127.0.0.1:8080
```

### 2. 生成API Key

```bash
python -c "import os; print(os.urandom(32).hex())"
```

将生成的密钥填入 `.env` 文件的 `API_KEY` 字段。

## 📡 API认证方式

所有API请求都需要提供API Key，有两种方式：

### 方式1: Header方式（推荐）
```http
X-API-Key: your-api-key-here
```

### 方式2: 查询参数
```http
POST /api/v1/devices?api_key=your-api-key-here
```

### WebSocket连接
```
ws://localhost:8080/ws?api_key=your-api-key-here
```

## ⚠️ 安全提示

1. **永远不要**将 `.env` 文件提交到代码仓库
2. 生产环境务必设置强 `API_KEY`
3. 生产环境设置 `ENV=production` 以禁用API文档
4. 建议使用HTTPS加密传输
5. 定期轮换API Key

## 📋 功能说明

### Web管理面板

| 标签页 | 功能 |
|--------|------|
| 屏幕控制 | 查看设备屏幕、点击、截屏、刷新 |
| 快速操作 | 系统按键(主页/返回/音量)、文字输入、滑动 |
| 脚本管理 | 创建、查看、执行、删除自动化脚本 |
| 脚本录制 | 录制手机操作、自动生成脚本、保存 |
| 操作日志 | 查看所有操作历史记录 |

### API接口

#### 设备管理
- `POST /api/v1/devices` - 获取设备列表
- `GET /api/v1/devices/{device_id}` - 获取设备详情
- `POST /api/v1/devices/{device_id}/connect` - 连接设备
- `POST /api/v1/devices/{device_id}/disconnect` - 断开设备

#### 设备控制
- `POST /api/v1/devices/{device_id}/tap` - 点击操作
- `POST /api/v1/devices/{device_id}/swipe` - 滑动操作
- `POST /api/v1/devices/{device_id}/input` - 输入文字
- `POST /api/v1/devices/{device_id}/key` - 按键操作
- `POST /api/v1/devices/{device_id}/screenshot` - 截图

#### 脚本管理
- `GET /api/v1/scripts` - 获取脚本列表
- `POST /api/v1/scripts` - 创建脚本
- `GET /api/v1/scripts/{script_id}` - 获取脚本详情
- `PUT /api/v1/scripts/{script_id}` - 更新脚本
- `DELETE /api/v1/scripts/{script_id}` - 删除脚本
- `POST /api/v1/scripts/{script_id}/execute` - 执行脚本

#### 脚本录制
- `POST /api/v1/record/start` - 开始录制
- `POST /api/v1/record/stop` - 停止录制
- `GET /api/v1/record/status` - 获取录制状态
- `POST /api/v1/record/action` - 录制动作

#### 日志查询
- `GET /api/v1/logs` - 获取操作日志

#### WebSocket
- `ws://localhost:8080/ws` - 实时通信

## 📱 前置条件

### 系统依赖
- **ADB**: Android Debug Bridge
- **Scrcpy**: 屏幕镜像工具 (可选，用于低延迟投屏)
- **Tesseract OCR**: 光学字符识别 (可选，用于文字识别)
- **FFmpeg**: 多媒体处理工具 (可选)

### Android设备设置
1. 开启"开发者选项"
2. 开启"USB调试"
3. 用USB连接手机或使用无线ADB
4. 在手机上授权USB调试

## 🎯 使用示例

### 1. 连接设备
```bash
# 先查看已连接设备
adb devices

# 如果是网络设备
adb connect 192.168.1.100:5555
```

### 2. 访问Web面板
打开浏览器访问: http://localhost:8080/

### 3. 选择设备
在左侧设备列表中点击要控制的设备

### 4. 开始使用
- 在"屏幕控制"标签页点击刷新查看屏幕
- 在"快速操作"标签页使用系统按键
- 在"脚本录制"标签页录制操作

## 🐛 常见问题

### Q: 找不到设备?
A: 确保手机已开启USB调试，并用USB连接或使用 `adb connect` 连接网络设备

### Q: 点击没反应?
A: 确保设备已在Web面板中选中，并且设备状态是"connected"

### Q: 如何录制脚本?
A: 1. 选择设备 2. 切换到"脚本录制"标签 3. 点击"开始录制" 4. 在手机上操作 5. 点击"停止录制" 6. 保存脚本
